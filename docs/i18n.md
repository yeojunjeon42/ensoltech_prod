# i18n (EN / KO)

A two‑language toggle, no library. Default is **Korean** (the primary
audience).

## How it works

`src/contexts/LanguageContext.tsx` exposes a tiny context:

```ts
type Language = 'en' | 'ko';

const { language, toggleLanguage } = useLanguage();
```

Components consume it and render conditionally:

```tsx
{language === 'en' ? englishCopy : koreanCopy}
```

The toggle button lives in `Navbar.tsx`. On desktop it sits to the
right of the nav. On mobile it lives at the top of the drawer as a
small pill.

## Where the copy lives

| Surface | Source |
|---------|--------|
| Hero intro paragraph | Hard‑coded constants in `src/components/Hero.tsx` |
| Contact form labels / placeholders / status | `COPY` object in `src/components/Contact.tsx` |
| Solution names + specs | `solutionData.ts` (`name` / `nameKorean`, `specifications` / `specificationsKorean`) |
| Nav category labels | `CATEGORIES` array in `Navbar.tsx` (`title` / `titleKorean`) |
| Footer | Mixed — Korean product list is hard‑coded; address is Korean‑only by intent |

## Adding new bilingual copy

Two patterns in use; pick whichever fits the size of the change.

**Tiny strings (1–2 words):** inline ternary.

```tsx
<span>{language === 'en' ? 'Home' : '홈'}</span>
```

**Anything longer or more than ~3 strings on one screen:** a `COPY`
object keyed by language, like the one in `Contact.tsx`. This keeps
both languages adjacent and obvious to a translator.

```ts
const COPY = {
  en: { title: 'Contact us', /* … */ },
  ko: { title: '문의하기',     /* … */ },
} as const;
const t = COPY[language];
```

Don't reach for `i18next` or `react-intl`. The site is two languages,
two screens, zero pluralisation; a context + ternaries is the right
size.

## Persistence

Today the toggle resets on full page reload (state is in memory). If
you add persistence, write to `localStorage` inside `LanguageProvider`
and read it back on mount — don't sprinkle storage calls across
consumers.

## Korean text rules of thumb

- Use the Korean text **as the primary string** in the data model
  field named `…Korean` even though the rest of the codebase is
  English. Don't romanise.
- Korean line lengths run shorter than English; if you tighten width,
  re‑check the English render.
- The `Footer.tsx` Korean product list is intentionally Korean‑only —
  do not translate it without confirming with the owner.
